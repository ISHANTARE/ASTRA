PROJECT OBJECTIVE

ASTRA analyzes satellite orbits and space debris to detect potential conjunction events.

The platform must:

• load orbital objects (~30,000 objects)
• propagate orbits using SGP4
• apply multi-stage filtering to reduce comparisons
• detect close approaches
• compute Time of Closest Approach (TCA)
• compute relative velocity at closest approach
• generate conjunction risk reports
• visualize orbital traffic in 3D.

SYSTEM ARCHITECTURE

ASTRA consists of three major subsystems:

1. Visualization System (Frontend)
   React + Three.js WebGL interface displaying satellites orbiting Earth.

2. Orbital Prediction Engine (Backend)
   SGP4 propagation producing object trajectories.

3. Orbital Analysis Engine (Backend)
   Filtering pipeline and conjunction detection algorithms.

All heavy computation must run **server-side**.

MANDATORY ORBITAL COMPUTATION RULES

Follow these rules strictly:

1. Use SGP4 via trusted Python libraries (`sgp4` or `skyfield`).
2. Never implement orbital mechanics equations manually.
3. Never approximate satellite motion using circular orbit formulas.
4. Handle coordinate frames properly (SGP4 outputs TEME).

Convert coordinates appropriately for visualization.

---

TRAJECTORY PRECOMPUTATION RULE

Propagation must occur **once per object across all simulation time steps**.

Never propagate objects inside pairwise comparison loops.

Correct architecture:

objects
→ propagate trajectories
→ store trajectory arrays
→ perform pairwise distance analysis

This prevents redundant propagation and drastically reduces computation.

---

SIMULATION PARAMETERS

Prediction window: 24 hours
Time resolution: 5 minutes
Total simulation steps: 288

All propagation and distance calculations must use these parameters.

---

MULTI-STAGE FILTERING PIPELINE

ASTRA must avoid full O(N²) comparisons across all objects.

The filtering pipeline is:

GLOBAL DATASET
↓
USER REGION FILTER
↓
ALTITUDE FILTER
↓
ORBITAL PLANE FILTER
↓
ORBIT INTERSECTION CHECK
↓
ORBITAL SPATIAL GRID INDEXING
↓
CANDIDATE PAIR GENERATION
↓
SGP4 PROPAGATION
↓
DISTANCE CALCULATION

Only candidate pairs should be analyzed.

---

CONJUNCTION ANALYSIS OUTPUT

Each detected event must include:

• object_1
• object_2
• closest_distance_km
• time_of_closest_approach_utc
• relative_velocity_km_s
• risk_classification

Relative velocity is computed as the magnitude of the difference between velocity vectors at TCA.
